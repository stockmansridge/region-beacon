# Customer-side Screen Parity Roadmap

Target screens for the visitor/customer experience, aligned to the GetStampd
mockup. Documented here so future UI work stays consistent — do NOT implement
all of these in one pass.

1. Welcome
2. Sign up
3. Passport progress
4. Trail Map
5. Wineries / Venues list
6. Check-in confirmation
7. Passport stamp grid
8. Rewards
9. Venue details
10. Offers
11. Invite friends
12. More menu

Notes:
- All demo screens live under `/demo/*` and must be clearly labelled as samples.
- Real visitor data, QR check-ins, leaderboard, and storage wiring are out of
  scope for these UI parity passes unless explicitly requested.
- Bottom nav across customer screens: Passport, Trail Map, Wineries, Rewards, More.
